Resumen de Requisitos Cubiertos

Req. Descripción Endpoints implementados OK
1 CRUD de Usuarios POST /api/usuarios, GET /api/usuarios/{id}, PUT /api/usuarios/{id}, DELETE /api/usuarios/{id}
2 CRUD de Platos GET /api/restaurantes/{id}/platos, POST, PUT /api/platos/{id}, DELETE /api/platos/{id}
3 CRUD de Restaurantes + propietario GET /all, GET /{id}, GET / (mis rest.), POST, PUT /{id}, DELETE /{id}
4 Pedidos (crear, historial, detalle) POST /api/pedidos, GET /api/usuarios/{id}/pedidos, GET /api/pedidos/{id}
5 Valoraciones + ordenar por media GET /api/restaurantes/{id}/valoraciones, POST, GET /all?sort=valoracionMedia
6 Búsqueda por nombre/localidad GET /api/restaurantes/all?q=texto
7 Filtrar y cambiar estado acepta pedidos GET /all?estado=acepta, GET /all?estado=no-acepta, PUT /{id}/estado
Extra POST /api/sesiones (+0.5 pts) POST /api/sesiones — Login con JSON, devuelve token Base64
Extra Categorías GET /api/categorias, POST /api/categorias (admin)
Auth HTTP Basic en todos los endpoints protegidos Autenticación Basic con Authorization: Basic <base64>
Errores Respuestas de error JSON con código HTTP correcto 401, 403, 404, 409, 400 — todos con body ErrorResponse
